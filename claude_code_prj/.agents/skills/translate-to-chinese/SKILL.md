# 英文翻译技能

将英文文章翻译成自然流畅的中文，同时保持专业性和准确性。

## 核心原则

**自然流畅**：译文应符合中文表达习惯，避免英文式的生硬翻译。长句拆分成多个短句，使阅读更流畅。

**专业术语处理**：
- 保留英文原文，格式为：中文翻译/English
- 首次出现时标注，后续可以只用中文
- 例如：机器学习/machine learning、应用程序接口/API

**不确定术语**：当某个术语有多种合理翻译时，在译文中直接标注所有选项，格式为：
- 选项1/选项2/选项3 (English)
- 例如：数据湖/数据池 (data lake)

## 输入处理

支持多种输入格式：
- **文本文件**（.txt, .md）：直接读取并翻译
- **文档文件**（.docx, .pdf）：提取文本后翻译
- **对话粘贴**：用户直接粘贴的文本内容

识别输入类型并相应处理。如果用户提供文件路径，先读取文件内容。

## 输出格式

始终使用双语对照格式，便于用户对比原文和译文：

```markdown
# [文章标题]

## [章节标题]

**原文：**
[英文原文段落]

**译文：**
[中文翻译段落]

---

**原文：**
[下一段英文]

**译文：**
[下一段中文]
```

## 翻译策略

**句子处理**：
- 长句（超过20个单词）拆分成2-3个短句
- 保持逻辑连贯，使用适当的连接词
- 避免逐字翻译，关注整体意思

**段落结构**：
- 保持原文的段落划分
- 必要时调整句子顺序以符合中文习惯

**语气和风格**：
- 技术文档：准确、简洁
- 学术论文：严谨、正式
- 商业文章：专业、易懂
- 根据原文风格调整译文语气

## 质量检查

翻译完成后，快速检查：
- 专业术语是否标注了英文原文
- 长句是否已拆分
- 译文是否自然流畅
- 不确定的术语是否给出了多个选项

## 输出文件

- 如果输入是文件，输出为同名的markdown文件（添加_translated后缀）
- 如果输入是对话文本，直接在对话中展示翻译结果
- 文件保存在原文件所在目录或用户指定位置

## 示例

**输入：**
"Machine learning algorithms can process vast amounts of data to identify patterns and make predictions."

**输出：**

**原文：**
Machine learning algorithms can process vast amounts of data to identify patterns and make predictions.

**译文：**
机器学习/machine learning 算法可以处理海量数据。它能识别其中的模式，并做出预测。
