# 五子棋项目问题清单

## 需要优先修复的问题

- 当前棋盘状态和游戏状态主要保存在前端内存中，刷新页面后对局会丢失，可以增加本地存储或重新开始确认机制。
- 落子、胜负判断、AI 行棋等逻辑分散在多个文件中，后续维护时容易出现状态不同步的问题，建议进一步抽离统一的状态管理模块。
- AI 计算在主线程执行，复杂局面下可能造成页面短暂卡顿，可以考虑使用 Web Worker 或限制搜索深度。
- 用户连续快速点击棋盘时，可能在状态切换边界触发重复操作，需要在落子处理中增加更严格的回合锁定。
- 目前缺少自动化测试覆盖，尤其是胜负判断、禁手边界、平局判断和 AI 选择逻辑，建议补充单元测试。

## 可改进体验

- 棋盘在小屏幕设备上的适配仍可优化，建议增加响应式尺寸和触控点击容错。
- 对局结束后的提示较简单，可以增加重新开始、复盘或高亮获胜五子的位置。
- AI 难度如果固定，用户缺少选择空间，可以增加简单、普通、困难等难度选项。
- 当前页面缺少无障碍语义和键盘操作支持，可以补充按钮标签、状态播报和键盘导航。
- 错误状态和异常输入缺少统一提示，建议增加用户可见的反馈信息。

## 工程质量建议

- 建议统一文件命名风格，避免下划线和短横线目录名混用造成维护困惑。
- 建议为核心逻辑增加中文注释，尤其是 AI 评分、搜索剪枝和胜负检测部分。
- 构建、类型检查和测试命令应在 README 中明确说明，方便新协作者快速验证修改。
